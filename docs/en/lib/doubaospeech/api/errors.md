## TTS 错误码

### 成功

| 码值 | 说明 |
|------|------|
| 3000 | 成功 |

### 客户端错误 (3xxx)

| 码值 | 说明 | 处理建议 |
|------|------|----------|
| 3001 | 参数错误 | 检查请求参数 |
| 3002 | 认证失败 | 检查 token |
| 3003 | 频率限制 | 降低请求频率 |
| 3004 | 余额不足 | 充值或检查套餐 |
| 3005 | 文本过长 | 缩短文本或使用异步接口 |
| 3006 | 音色不存在 | 检查 voice_type |
| 3007 | 无权访问 | 检查资源权限 |

### 服务端错误 (5xxx)

| 码值 | 说明 | 处理建议 |
|------|------|----------|
| 5000 | 服务内部错误 | 重试或联系支持 |
| 5001 | 服务暂时不可用 | 稍后重试 |
| 5002 | 服务超时 | 重试 |

### 通用

| 码值 | 说明 |
|------|------|
| 0 | 成功 |
| -1 | 未知错误 |

### 任务状态

| 码值 | 说明 |
|------|------|
| 100 | 任务排队中 |

## HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 认证失败 |
| 403 | 无权限 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |

## 错误响应格式

```json
{
  "code": 3001,
  "message": "invalid parameter: text is empty",
  "reqid": "xxx"
}
```

## 错误处理建议

1. **认证错误**：检查 token 是否正确、是否过期
2. **参数错误**：检查必填参数，验证参数格式
3. **频率限制**：实现指数退避重试
4. **服务错误**：记录 reqid，联系技术支持
